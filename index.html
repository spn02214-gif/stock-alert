PROBLEM DEFINITION

01
전종목 수동 모니터링 불가
한국 KOSPI+KOSDAQ 약 2,400종목, 미국 나스닥100까지 포함하면 사람이 매시간 직접 확인하는 것은 구조적으로 불가능하다.
02
탐지→판단 사이 수작업 구간
조건에 맞는 종목을 찾는 과정 자체가 반복 노동이다. 판단은 사람이 해야 하지만, 탐지는 자동화 가능한 영역이다.
03
핵심 병목 구간
전체 프로세스에서 자동화 가능한 구간은 탐지 → 알림 전구간이다. 매수 판단은 여전히 사람의 영역으로 남긴다.
04
PC 의존 구조의 한계
로컬에서 실행하면 PC가 꺼질 때마다 시스템이 멈춘다. 장중 모니터링은 클라우드 독립 실행이 필수 조건이다.

03

ENTRY CONDITIONS

1
SHARP DECLINE
전일 대비 -5% 이상 하락
단기 급락 발생 포착. 반등 가능성이 있는 과매도 구간 진입 신호.
2
BULLISH CANDLE
당일 양봉 (종가 > 시가)
하락 중 반등 시작 확인. 급락 후 매수세 유입 신호.
3
VOLUME SURGE
거래량 ≥ 5일 평균 × 2배
수급 진입 확인. 거래량 동반 없는 반등은 신뢰도 낮음.
4
ABOVE MA20
현재가 > 20일 이동평균
추세 안에서의 조정 확인. 하락추세 종목 필터링.
# 4개 조건 AND 구조 — 전부 충족해야 알림 발송
if (
    decline >= 0.05            # 전일 대비 -5% 이상
    and close > open_           # 양봉
    and volume >= avg_vol * 2  # 거래량 2배 이상
    and close > ma20            # 20일선 위
):
    send_telegram_alert(ticker)
04

DESIGN DECISIONS

결정 항목 선택 대안 선택 근거
스캔 주기 1시간 10분, 30분 2~3일 스윙에 충분. GitHub 무료 한도(2,000분/월) 내 여유 확보. 노이즈 감소.
실행 환경 GitHub Actions PythonAnywhere, AWS Lambda 무료, PC 독립, 외부 API 제한 없음. 설정 복잡도 최소.
한국 데이터 pykrx 증권사 API (키움 등) 무료, 별도 인증 불필요, KOSPI+KOSDAQ 전종목 커버.
미국 종목 범위 나스닥100 S&P500, 전종목 변동성 크고 스윙 적합. 100개라 속도 빠름. S&P500은 대형주 위주로 변동성 낮음.
알림 채널 텔레그램 카카오톡, 이메일 Bot API 설정 가장 단순. 즉시 수신. 무료.
거래량 기준 5일 평균 × 2배 × 1.5배, × 3배 노이즈와 민감도의 균형점. 너무 낮으면 오신호, 너무 높으면 탐지 누락.

07

KEY LEARNINGS

LEARNING 01
요구사항 → 수치 구조화
"반등 신호"라는 추상적 개념을 4개의 정량 조건으로 분해했다. PM의 역할은 모호한 목표를 판단 가능한 기준으로 만드는 것이다.
LEARNING 02
비용·효율 균형 설계
무료 인프라 제약 안에서 커버리지를 최대화했다. 스캔 주기를 10분에서 1시간으로 조정한 것은 기술 결정이 아니라 목적 기반 설계 결정이다.
LEARNING 03
기획에서 구현까지 완결
조건 설계에서 끝내지 않고 Claude Code로 실제 작동하는 시스템까지 완성했다. 작동하는 결과물이 포트폴리오의 증거가 된다.
